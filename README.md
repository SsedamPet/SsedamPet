## 📝 프로젝트 소개

### 🎯 기획 배경

> 💡 **반려동물 건강 데이터의 체계적 기록 관리, 그 이상의 가치를 제공합니다.**

- 📊 반려동물 건강 데이터(음수량, 식사량 등)의 체계적 기록 관리의 어려움 해결
- 🔍 단순 기록을 넘어 **데이터 기반의 분석 가이드**를 제공하는 서비스 필요성 증대

<br/>

### 👥 대상 사용자

<div align="center">

| 구분 | 설명 |
|:---:|---|
| 🩺 **건강 관리** | 반려동물의 건강 상태를 매일 체계적으로 체크하고 싶은 보호자 |
| 💬 **커뮤니티** | 다른 반려인들과 일상을 공유하고 실시간으로 소통하고 싶은 사람 |
| 🤖 **AI 상담** | 이상 징후 발생 시 AI를 통한 건강 상담이 필요한 사람 |

</div>

<br/>

### ✨ 기대 효과

| 효과 | 내용 |
|---|---|
| 📈 데이터 시각화 | 주간 평균 비교 분석을 통해 컨디션 변화를 조기에 감지 |
| 🎯 원스톱 서비스 | 기록, 분석, 상담, 커뮤니티를 하나의 플랫폼에서 해결 |
| ⚡ 실시간 상호작용 | SSE 기술을 통한 실시간 알림으로 커뮤니티 활성화 유도 |

## 📐 컨벤션

### 커밋 컨벤션

| 태그 | 설명 |
|---|---|
| `feat` | 새로운 기능 추가 |
| `fix` | 버그 수정 |
| `docs` | 문서 수정 (README.md 등) |
| `refactor` | 코드 리팩토링 (기능 변경 없음) |
| `style` | 코드 포맷팅, 스타일 변경 (로직 변경 없음) |
| `chore` | 빌드 업무, 패키지 설정 등 사소한 변경 |

### 코드 명명 규칙

| 구분 | 규칙 |
|---|---|
| Backend - 클래스명 | PascalCase |
| Backend - 변수/메서드명 | camelCase |
| Backend - 상수명 | UPPER_CASE |
| Frontend - 컴포넌트명 | PascalCase |
| Frontend - 핸들러 함수 | onXxxHandler |
